## 3. 文档规划 vs 实际实现 — 关键差异汇总

| #  | 文档规划                     | 实际实现                                      | 原因                                                                                           |
| -- | ------------------------ | ----------------------------------------- | -------------------------------------------------------------------------------------------- |
| 1  | SetParent 嵌入任务栏子窗口       | 独立浮动窗口覆盖在任务栏上方                            | Win11 任务栏结构变化，SetParent 方案不稳定；TranslucentTB 等成熟项目均使用浮动方案                                     |
| 2  | C++20                    | C++17                                     | 项目实际设定为 C++17，功能上无影响                                                                         |
| 3  | SetDrawingEffect 逐字着色    | PushAxisAlignedClip 裁剪方案（已修复居中对齐问题）       | 文档自身提到的备选方案，实现更简单可靠；已修复居中对齐时高亮位置错误的 bug                                                      |
| 4  | 绑定模式 + 进程监控              | **Native Host 托管模式**（v0.4.0）              | ProcessMonitor 已移除，改用 MoeKoeMusic `moekoe_native_hosts` + `nativeHostManager.js` 托管 EXE 生命周期 |
| 5  | WS\_EX\_TRANSPARENT 鼠标穿透 | 不使用 WS\_EX\_TRANSPARENT                   | 需要接收鼠标消息以支持悬停控制按钮和拖动                                                                         |
| 6  | 日志写入 %TEMP%/硬编码          | 日志写入 exe 同级 debug.log（可移植）                | 调试阶段方案，使用 GetModuleFileName 获取 exe 目录，避免硬编码路径                                                |
| 7  | "解除绑定"菜单项                | 无此菜单项                                     | 绑定模式未实现                                                                                      |
| 8  | 手动编辑 JSON 配置             | D2D 原生 GUI 设置界面                           | 已完整实现配置 GUI，包含颜色选择器、滑块、字体选择按钮等                                                               |
| 9  | 无拖动调整位置                  | 支持拖动歌词窗口调整位置并保存到配置                        | 用户需求新增功能                                                                                     |
| 10 | 无 Z-order 保持逻辑           | 三重 TOPMOST 防护（创建/消息/周期性）                  | 修复点击任务栏/展开托盘时歌词被覆盖的问题                                                                        |
| 11 | TerminateThread 强制终止线程   | 协作式 join() 退出                             | 代码审查发现的安全隐患，全部 3 处（ws/http/process）已移除                                                       |
| 12 | 散落各处的魔数                  | constants.h 集中管理                          | 代码审查建议，提取端口/尺寸/消息号/UI参数等为命名常量                                                                |
| 13 | 无异常恢复机制                  | WM\_TIMER catch 后尝试 Shutdown+Reinitialize | 代码审查建议，防止渲染器异常导致程序卡死                                                                         |
| 14 | 无配置值校验                   | std::clamp 范围验证                           | 代码审查建议，防止用户手动编辑 JSON 导致非法值                                                                   |
| 15 | 无长歌词滚动                   | 跑马灯 6 状态机（bounce/loop/off）                | 设计文档需求：短歌词居中不动、长歌词往返滚动、超长加速                                                                  |
| 16 | 无 API 自动开启               | api\_enabler 模块自动检测/修改/重启                 | 用户需求：WS 连接失败时自动开启 MoeKoeMusic 的 API 模式                                                       |
| 17 | 图标取最小尺寸                  | manifest icons 128 排第一                    | MoeKoeMusic 按 Object.keys()\[0] 取图标，需将最大尺寸放首位                                                |
